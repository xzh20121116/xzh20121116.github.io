07 排查堆内存溢出

阅读量: 今晚打老虎 2020-07-21 18:49:54

模拟堆内存溢出

  1. 新建一个 Oom.java 文件，内容如下

    import java.util.ArrayList;
import java.util.List;

public class Oom {
    public static void main(String[] args) {
      // 根据各自虚拟机配置设定，这里是 64m 大小
        int size = 1024 * 1024 * 8 * 8;
        List<byte[]> list = new ArrayList<>();
        int times = 0;
        for (;;){
            times++;
            System.out.println("第  "+times+"  次执行");
            list.add(new byte[size]);
        }
    }
}

  2. 在 Oom.java 文件路径执行，来生成 Oom.class 文件

     javac Oom.java

  3. 运行 Oom.class 文件，观察终端，发现执行大约 6-8 次时发生异常

    java -XX:+HeapDumpOnOutOfMemoryError -Xms512m -Xmx512m Oom

    -XX:+HeapDumpOnOutOfMemoryError：是用来设置，发生 oom 时自动生成 dump 文件

    -Xms512m -Xmx512m：用来指定程序的内存大小，这里是这个设置是为了让程序快速报错

这三步执行完之后，在指定路径下会有三个文件，如下图所示

其中 .hprof就是我们要分析的 dump 文件

通过 Mat 分析 Dump 文件

下载地址

https://www.eclipse.org/mat/downloads.php

运行 Mat

下载完之后，如果是个压缩包，需要解压缩

即使解压缩也无法直接运行

启动方式 1

  1. 选中 Mat.app

  2. 右键-显示包内容-双击图中红框文件

启动方式 2

/Applications/mat.app/Contents/MacOS/MemoryAnalyzer

选择要分析的 dump 文件

  1. 启动之后，出现如图所示界面；

  1. 点击【open a Heap Dump】，选择教程第一步生成的.hprof文件，选择完之后，出现如下图

  2. 点击图中的 【Leak Suspects】（两个红框，是一样，随意点击一个），点击之后，出现如下图所示
