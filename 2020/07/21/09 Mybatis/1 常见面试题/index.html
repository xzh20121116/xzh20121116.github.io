${}和#{}的区别是什么

${}

是 properties 文件中的变量占位符，它可以用于标签属性值和 sql 内部，属于静态文本替换。

比如${driver}会被替换为com.mysql.jdbc.Driver.

#{}

是 sql 的参数占位符，会将 sql 中的#{}替换为？

在 sql 执行前会使用 PreparedStatement 的参数设置方法，按序给 sql 的？占位符设置参数值。

比如：ps.setInt(o,parameterValue)

#{item.name} 的取值方式为使用反射从参数对象中获取 item 对象的 name 属性值，相当于 param.getItem.getName（）；

dao 接口的原理

最佳实践中，通常一个 xml 映射文件，都会写一个 Dao 接口与之对应，请问，这个 Dao 接口的工作原理是什么？Dao 接口里的方法，参数不同时，方法能重载吗？

Dao 接口，也就是人们常说的 Mapper 接口，

接口的全限名，就是映射文件中的 namespace 的值，

接口的方法名，就是映射文件中的MappedStatement 的 id 值

接口方法内的参数，就是传递给 sql 的参数。

Mapper 接口是没有实现类的，当调用接口方法时，接口全限名+方法名拼接字符串作为 key 值，可唯一定位一个 MappedStatement，举例：com.d1mq.mappers.UserDao.findStudentById,可以找到 namespace 为com.d1mq.mappers.UserDao下面 id= findStudentById 的 MappedStatement。

在 mybaits 中，每一个<select>、<insert>、<update>、<delete>标签都会被解析为一个 MappedStatement 对象

Dao 接口里的方法是不能重载的，因为是全限名+方法名的保存和寻找策略。

Dao 接口的工作原理是 jdk 动态代理，Mybatis 运行时会使用 JDK 动态代理为 Dao接口生成 proxy 对象，代理对象proxy会拦截接口方法，转而执行 MappedStatement 所代表的的 sql，然后将 sql 执行结果返回。

Mybatis 是如何进行分页的

mybatis 使用 RowBounds 对象进行 分页，它是针对 ResultSet 结果集执行的内存分页，而非物理分页。

可以在 sql 内直接写带有物理分页逻辑的参数来完成物理分页功能，也可以使用分页插件来完成物理分页

分页插件的原理是什么？

基本原理是使用 Mybatis 提供的插件接口，实现自定义插件，在插件的拦截方法内拦截待执行的 sql，然后重写 sql，根据 dialect 方法，添加对应的屋里分页语句和屋里分页参数

举例：

select	xx from	user

拦截sql 后重写为

 select t.xx from (select \* from student ) t limit 0,10

Mybatis 值可以编写针对 ParameterHandler、ResultSetHandler、StatementHandler、Executor这 4种接口的插件

Mybatis 使用 jdk 的动态代理，为需要拦截的接口生成代理对象以实现接口方法拦截功能，每当执行这 4 中接口对象的方法是，就会进入拦截方法，具体就是 InvocationHandler 的 invoke()方法，当然只会拦截你指定的要拦截的方法

能，jdbc 都能，mybatis 当然也能

动态 sql 有哪些？9 个

<trim>、<where>、<set>、<foreach>、<if>、<choose>、<when>、<otherwise>、<bind>、

动态 sql 的做什么用的？

可以让我们在 xml 映射文件中，以标签的形式编写动态 sql，完成逻辑判断和动态拼接 sql 的功能

动态 sql 的执行原理（简述）

使用 OGNL 从 sql 参数对象中计算表达式的值，根据表达式的值动态拼接 sql，一次来完成动态 sql 的功能

是如何将sql 执行结果封装成目标对象并返回的？

第一种：

使用<resultMap>标签，逐一定义列明和对象属性名之间的映射关系

第二种

使用 sql 列的别名，将列名别名书写为对象的属性名，mybatis 会自动忽略列名的大小写

有了 列名与属性的映射关系后，mybatis 通过反射创建对象，同时使用反射个对象的属性逐一赋值并返回，哪些招不到映射关系的属性，是无法完成赋值的

mybatis 能执行一对多，多对多的关联查询？有哪些实现方式？一区他们直接的区别

能，支持一对一、一对多、多对一、多堆多

多对一查询，其实就是一对一查询，只要把 selectOne()修改为 selectList()即可。

多对多查询，其实就是一对多查询，只要把 selectOne()修改为 selectList()即可。

关联对象查询，有两种实现方式，一种是单独发送一个 sql 去查询关联对象，赋给主对象，然后返回主对象。另一种就是使用嵌套查询，嵌套查询的含义为使用 join 查询，一部分列是 a 对象的属性值，另外一部分列是关联对象 b 的属性值，好处是只发一个 slq 查询，就可以把助对象和其关联对象查出来。

那么问题来了，join 查询出来 100 条记录，如何确定助对象是 5 个，而不是 100 个？

其去重的原理是<resultMap>标签内的<id>子标签，指定了唯一确定一条记录的 id 列，mybatis 根据列值来完成 100 条记录的去重复功能，<id>可以有多个，代表了联合主键的语意

同样助对象的关联对象，也是根据这个原理去重复的，尽管一般情况下， 只有助对象会有重复记录，关联对象一般不会重复

延迟加载的实现原理是？

Mybatis 仅支持 association 关联对象和 collection 关联集合对象的延迟加载，association 指的就是 一对一，collection 指的就是一对多查询

在 mybatis 的配置文件中可以配置是否启动延迟架子啊

lazyLoadingEnabled =true

原理

使用 cglib 创建目标对象的代理对象，当调用目标方法时，进入拦截器方法，比如调用 a.getB().getName，拦截器发现 a.getB()是 null 值，那么就会单独发送事先保存好的关联查询 b 对象的 sql，把 b 查询上来，然后调用 a.setB（）,于是 a 的对象 b 就有值了，接着完成 a.getB().getName（）,这就是延迟加载的基本原理

不光是 mybaits，几乎所有的，包括 hibernate，支持延迟加载的原理都是一样的。
