类加载过程

阅读量: 今晚打老虎 2020-06-22 10:00:32

在冯。诺依曼的计算机模型中，任何程序都需要加载到内存才能与cpu进行交流。

字节码.class文件同样需要加载到内存中，才可以实例化。“兵马未动，粮草先行”。ClassLoader 就是准备粮草的先行军，他的使命就是「提前加载.class类文件到内存中」，在加载类时，使用的是『Parents Delegation Model』，“双亲委派模型”，更合适的叫法是"溯源委派加载模型"

1. 类的过程分为三步走：

1.1. load（加载）

读取类文件，产生二进制流，并转化为特定的数据结构（字节流所代表的静态存储结构转换为方法区的运行时数据结构），初步校验cafe babe魔法数、常量池、文件长度、是否有父类等，然后创建对应类的java.lang.Class实例

1.2. link（连接）

这个阶段又分为以下三步走：

验证：是更详细的校验，比如final是否合规，类型是否正确，静态变量是否合理等；

准备：这个阶段是为静态变量分配内存，并设定默认值

解析：解析类和方法，确保类与类之间的相互引用正确性，完成内存结构布局

1.3. init（初始化）

执行类构造器方法，如果赋值运算是通过其他类的静态方法来完成的，那么会马上解析另外一个类，在虚拟机栈中执行完毕后通过返回值进行赋值

2. 什么是类加载器

类加载是一个将.class字节码文件实例化成java.lang.Class对象并进行相关初始化的过程。这个过程中，jvm会初始化继承树上还没有被初始化过的所有父类，并且执行这个链路上所有未执行过的静态代码块、静态变量赋值语句等。

全小写的class是关键字，用来定义类

首字母大写的CLass，他是所有class的类，就像数值型的数据类型都是number，所有的类都是CLass

类加载器类似于原始部落结构，存在权利等级制度。最高的一层是家族中威望最高的bootstracp。他是在jvm启动时创建的，通常由于操作系统相关的本地代码实现，是最根基的类加载器，负责装在最核心的java类，比如Object、System、String等；

第二层是在jdk9中，称为Platform ClassLoader，即平台类加载器，用来加载一些扩展的系统类，比如xml，加密、压缩相关的功能类等，jdk9之前的加载器是Extension CLassLoader；

第三层是Application ClassLoader 应用类加载器，主要是加载用户自定义的CLassPath路径下的类

第二三层的由java语言实现的，用户也可以自定义类加载器。

Bootstracp 是通过c/c++实现的，并不存在于jvm体系内，

这三层有等级，并不是继承关系

3. 类加载过程

低层次的类加载器不能覆盖更高层次类加载器已经加载的类。如果低层次的类加载器想加载一个未知类，需要非常礼貌的向上级逐级询问：“请问，这个类已经加载了吗？”

被询问的高层次加载器会自问两个问题：“第一，我是否已经加载过此类？第二，如果没有，我是否可以加载此类”，只有在所有的高层次类加载器回答都为“否”时，才可以让当前类加载器加载这个未知类。
