事务实现原理

阅读量: 今晚打老虎 2020-05-31 20:43:04

1. spring 的五种事务隔离级别

  1. 默认使用数据库的事务
  2. 读未提交
  3. 读已提交
  4. 可重复读
  5. 串行化

2. spring 的七种传播属性

  1. required：最常用，支持当前事务，如果没有，就新建
  2. supports：支持当前事务，如果没有，就没有
  3. mandatory：支持当前事务，如果没有，抛异常
  4. requires_new：新建事务，如果当前有，则挂起当前事务
  5. not_supports：非事务执行，如果有则挂起
  6. never：非事物执行，如果有，抛异常
  7. nested：如果有，则嵌套在事务内执行，如果没有，就新建

3. 实现原理

spring 是通过 AOP 来实现声明式事务

spring 对方的增强有五种方式：

  1. 前置增强–BeforeAdvice：目标方法执行前增强
  2. 后置增强–AfterReturningAdvice：目标方法执行后增强
  3. 环绕增强–MethodInterceptor：目标方法执行前、后都增强
  4. 异常抛出增强–ThrowsAdvice：目标方法抛出异常后执行增强
  5. 引介增强–IntroductionInterceptor：为目标类添加新的属性和方法

声明式事务的实现就是通过环绕增强的方式，在目标方法执行前开启事务，在目标方法执行后提交或者回滚事务。

以注解方式为例子：

  1. 在相关的类和方法上添加@Transactional
  2. 在代码运行时生成一个代理对象，根据@Transactional的属性配置信息，决定目标方法是否由TransactionInterceptor拦截
  3. 由TransactionInterceptor拦截的，会在目标方法开始执行前创建并加入事务，执行目标方法的逻辑，根据执行情况，由抽象事物管理器AbstractPlatformTransactionManager操作数据源 DataSource 提交或回滚事务

4. @Transactional

该注解可以配置到方法或类上

如果两者同时配置，以方法上的配置为准

属性名 说明
name 当在配置文件中有多个 TransactionManager 时，可以使用该属性指定选择哪个事务管理器
propagation 事务的传播界别，默认为 Required
isolation 事务的隔离度，默认值采用 Default
timeout 事务的超时时间，默认值为-1.如果超过该时间但是事务没有完成，则自动回滚事务
read-only 指定事务是否为只读事务，默认值为 false。为了忽略那些不需要事务的方法，比如读数据，可以设置为 true
rollback-for 用于指定能够触发回滚的异常类型，多个异常类型用逗号分隔
no-rollback- for 指定，不需要回滚的异常

5. Aop 的自调用问题

spring 的 aop 代理下，只有目标方法由外部调用，目标方法才由 spring 生成的代理对象来管理

内部的自调用，会导致 aop 失效，因为spring 操作的都是代理对象，自调用是本身，spring 无法管理

判断一个Bean是否是AOP代理对象可以使用如下三种方法：
AopUtils.isAopProxy(bean) ： 是否是代理对象；
AopUtils.isCglibProxy(bean) ： 是否是CGLIB方式的代理对象；
AopUtils.isJdkDynamicProxy(bean) ： 是否是JDK动态代理方式的代理对象

5.1. 解决方案

5.1.1. 从 beanFactory 中获取对象

所有的对象都可以从 BeanFactory 中获取，那么当你想调用自己的时候，也从 BeanFactory 获取自己

public void insert02(User u){
  getService().insert01(u);
}
private UserService getService(){
  return SpringContextUtil.getBean(this.getClass());
}

5.1.2. 获取代理对象

private UserService getService(){
  // 采取这种方式的话，
  //@EnableAspectJAutoProxy(exposeProxy=true,proxyTargetClass=true)
  //必须设置为true
  return AopContext.currentProxy() != null ? (UserService)AopContext.currentProxy() : this;
}

5.1.4. 通过 BeanPostProcessor

通过 BeanPostProcessor 在目标对象中注入代理对象：

Beanfactory 中的每个 bean 初始化完毕后，调用BeanSelfAware的setSelf方法，把自己的代理对象注入自身

public class InjectBeanSelfProcessor implements BeanPostProcessor  
{  
   
    public Object postProcessAfterInitialization(Object bean, String beanName) throws BeansException  
    {  
        if(bean instanceof BeanSelfAware)  
        {  
            System.out.println("inject proxy：" + bean.getClass());  
            BeanSelfAware myBean = (BeanSelfAware)bean;  
            myBean.setSelf(bean);  
            return myBean;  
        }  
        return bean;  
    }  
   
    public Object postProcessBeforeInitialization(Object bean, String beanName) throws BeansException  
    {  
        return bean;  
    }  
}
