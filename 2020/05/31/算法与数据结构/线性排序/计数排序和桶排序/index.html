计数排序和桶排序

阅读量: 今晚打老虎 2020-05-31 20:43:04

快排，堆排的时间复杂度是 nlogn

无论冒泡还是快排、堆排，都是基于元素之间的比较来进行排序的

冒泡：

因为 8>3，所以8 和 3 的位置交换

堆排：

因为 10>7，所以 10 和 7 的位置交换

有一些特殊的排序并不基于元素比较，如计数排序、桶排序、基数排序等

以计数排序来说，这种排序是利用数组下标来确定元素的正确位置的

1. 初识计数排序

1.1. ¶是什么

元素下标如何来帮助排序呢？

举个例子：数组中有 20 个随机整数，取值范围0-10，要求用最快的速度把这 20 个整数从小到大进行排序

**思考：**考虑到这些整数之智能在 0-10，这 11 个数中取值，取值范围有限，所以可以根据这有限的范围，建立一个长度为 11 的数组，数组下标从0-10，元素初始值全为 0

假设 20 个随机整数的值如下所示

9，3，5，4，9，1，2，7，8，1，3，6，5，3，4，0，10，9，7，9

下面就开始遍历这个无序的随机数组，当这个数与 index 相等时，对应 index 的值+1

例如，第一个整数是 9，那么 index 为 9 的值+1

第二个整数是 3，那么index 为 3 的值加 1

以此类对，最终统计完的数组如下图

此时，输出的数组已经是有序的了，在取值范围不大的情况下，性能超过 nlogn 的排序

1.2. ¶代码实现

public class 计数排序 {
	public static void main(String[] args) {
		int[] arr = {5, 2, 4, 3, 1, 7, 9, 8, 6, 5, 5, 5, 3, 3};
		countSort(arr);
	}

	static void countSort(int[] arr) {
		//得到数组的最大值
		int max = arr[0];
		for (int i = 1; i < arr.length; i++) {
			max = Math.max(max, arr[i]);
		}
		//根据最大值确定数组的长度,加 1 是因为 index 是从 0 开始的
		int[] countArr = new int[max + 1];
		// 遍历 要排序 的数组,统计每个 index 出现的次数,填充到对应的位置
		for (int value : arr) {
			countArr[value]++;
		}
		System.out.println(Arrays.toString(countArr));
    //[0, 1, 1, 3, 1, 4, 1, 1, 1, 1]
		// 遍历统计后的数组,重新排序
		int index = 0;
		int[] sortArr = new int[arr.length];
		for (int i =0;i<countArr.length;i++){
			for (int  j = 0;j<countArr[i];j++){
				sortArr[index++]=i;
			}
		}
		System.out.println(Arrays.toString(sortArr));
    //[1, 2, 3, 3, 3, 4, 5, 5, 5, 5, 6, 7, 8, 9]
	}
}

2. 计数排序的优化

2.1. ¶空间浪费问题

上面代码存在的问题

  1. 只以数组的最大值来决定数组的长度，并不严谨，如果范围是 95，96，99，那么如果创建长度为 100 的数组的话，是很大的空间浪费

2.2. ¶空间浪费问题-解决办法

不再以数组的最大值作为数组长度，而是以最大值最小值之间的差值+1 作为数组长度，Max - min + 1 作为数组的长度

同时，min 作为一个偏移量，用于计算整数在数组中的下边，即：96 的下标为 96-95（min）为 1，99 的下标为 99 - 95（min）为 4。

2.3. ¶值相同，区分先后顺序

举个例子：

给出一个成绩，按照成绩从高到低排序，如果分数相同，则按照原表的顺序排列，

姓名 成绩
小灰 90
小黄 99
小红 95
小白 94
小绿 95

以这个成绩表为例，稍作变形，变成下面的样子

如何变形的呢？

从统计数组的第二个元素开始，每个 index 的值，都是当前 index 值加上前一个 index 的值

index[i] =index[i]+index[i-1]

为什么要相加呢

这样相加之后，index 存储的值，就是对应的整数最终的排序的位置

例如，下标是 9 的元素值为 5，代表原始数组的整数 9，最终排在第五位

优化后的步骤如下：

2.4. ¶步骤

  1. 遍历数组，得到最大值和最小值，从而得出这俩的差值

    差值的作用：排序数组的大小=差值+1

  2. 统计数组

    创建一个统计数组，统计对应 index 出现的次数

  3. 统计数组变形

    遍历统计数组
使 index[i] =index[i]+index[i-1]

  4. 排序数组

    创建排序数组，长度和待排序数组一致，从后向前遍历待排序数组

2.5. ¶代码实现

public class 计数排序_优化版 {
	public static void main(String[] args) {
		int[] arr = {90,99,95,94,95};
		countSort(arr);
	}

	static void countSort(int[] arr) {
		//1. 得到数组的最大值和最小值,计算出差值 d
		int max = arr[0];
		int min = arr[0];
		for (int i = 1; i < arr.length; i++) {
			max = Math.max(max, arr[i]);
			min = Math.min(min, arr[i]);
		}
		int d = max - min;
		//2. 创建统计 数组 并统计对应 index 出现的次数
		int[] countArr = new int[d + 1];
		for (int value : arr) {
			// value 原始数组中的实际值,减去 min,才是数组的下标
			countArr[value - min]++;
		}
		System.out.println(Arrays.toString(countArr));
    //[1, 0, 0, 0, 1, 2, 0, 0, 0, 1]
		// 3. 统计数组做一下变形,index[i] =index[i]+index[i-1]
		// 从下标 1 开始,如果从 0 开始的话,0-1=-1,会越界
		for (int i = 1; i < countArr.length; i++) {
			//变形后的统计数组 的 值代表 最终的排序位置,1 就是第一位,2 就是第二位
			countArr[i] += countArr[i - 1];
		}
		// 4.倒序遍历原始数组,从统计数组找到正确位置,输出到结果数组
		int[] sortArr = new int[arr.length];
		for (int i = arr.length - 1; i >= 0; i--) {
			/**
			 * arr[i] :代表原始数组的最后一个元素的值
			 * arr[i] - min :代表排序统计数组中的下标
			 * countArr[arr[i] - min ] :该 index 对应值即为最终排序号
			 * countArr[arr[i] - min ]- 1 : 下次再遇到相同分数的时候,排序号减 1,即名次+1
			 */
			sortArr[countArr[arr[i] - min ]- 1] = arr[i];
			countArr[arr[i] - min]--;
		}
		System.out.println(Arrays.toString(sortArr));
    //[90, 94, 95, 95, 99]
	}
}

2.6. ¶时间复杂度

2.6.1. ¶结论：o（n+m）

2.6.2. ¶分析

n：原始数组的规模

m：最大值和最小值的差值

代码 1、2、4 都设计遍历原始数组，运算量都是 n

第 3 步遍历统计数列，运算量是 m，所以总体运算量是3 n+m,去掉系统，即o（n+m）

2.7. ¶为什么很少使用计数排序

  1. 当数组最大值和最小值差距过大时，不适用

    给 20 个整数，范围 1-1亿，如果使用计数排序，需要创建长度为 1 亿的数组，空间浪费

  2. 当数组值不是整数时，也不适合计数排序，无法创建对应的统计数组

对于以上的局限性，另一种线性时间排序算法做出了弥补，即：桶排序

2.8. ¶不适用的场景

数据范围比要排序的数据大很多，数据范围 1-20 亿，要给其中的 10 个数排序，那么就不适合

3. 什么是桶排序

类似于计数排序所创建的统计数组，桶排序需要创建若干个桶来协助排序

3.1. ¶什么是所谓是"桶？"

每一个桶（bucket）代表一个区间范围，里面可以承载一个或多个元素

给手机号排序就不适用，11 位，太大了

3.2. ¶工作原理

假如有一个 非整数 数组如下

{4.5, 0.84, 3.25, 2.18, 0.5}

看看桶排序的工作原理

  1. 创建这些桶，并确定每一个桶的区间范围

    具体需要创建多少桶，如何确定桶的区间范围，有很多种方式，这里创建的 桶的数量 等于 原始数组的数量，除了最后一个桶包含最大值之外，前面的各个桶的区间 按照比例来确定

    区间跨度 = （最大值-最小值）/（桶的数量-1）

  2. 遍历原始数组，把元素对号入座放入桶中

  1. 对每个桶的内部分别进行排序（这里只有第一个桶需要排序）

  2. 遍历所有的桶，输出所有元素

    0.5，0.84，2.18，3.25，4.5

到此为止，排序结束

3.3. ¶代码实现

public class 桶排序 {
	static void bucketSort(double[] arr) {
		//1.得到数列的最小值和最大值,并计算出差值 d
		double max = arr[0];
		double min = arr[0];
		for (int i = 1; i < arr.length; i++) {
			max = Math.max(max, arr[i]);
			min = Math.min(min, arr[i]);
		}
		double d = max - min;

		//2. 初始化桶
		int bucketNum = arr.length;
		ArrayList<LinkedList<Double>> bucketList = new ArrayList<LinkedList<Double>>(bucketNum);
		for (int i = 0; i < bucketNum; i++) {
			bucketList.add(new LinkedList<Double>());
		}

		// 3.遍历原始数组,将每个元素放入桶中
		for (double v : arr) {
			int num = (int) ((v - min) * (bucketNum - 1) / d);
			bucketList.get(num).add(v);
		}

		// 4.对每个桶内部进行排序
		double[] sortArr = new double[arr.length];
		int index = 0;
		for (LinkedList<Double> list : bucketList) {
			// jdk 底层采用了归并排序或归并更优化的版本
			Collections.sort(list);
			for (double element:list){
				sortArr[index]=element;
				index++;
			}
		}

		System.out.println(Arrays.toString(sortArr));
	}
}

在上述代码中，所有的桶都被保存在 ArrayList 集合中，每个桶都被定义成一个链表，这样便于在尾部插入元素

3.4. ¶时间复杂度

3.4.1. ¶结论on

3.4.2. ¶分析

假设有 n 个元素，分成 m 个桶

  1. 求最大值、最小值，运算量为 n
  2. 创建空桶，运算量为 n
  3. 把原始数组分到桶中，运算量为 n
  4. 在每个桶的内部做排序，在元素相对均匀的情况下，所有桶的运算量之和尾 n

桶排序的性能冰粉绝对稳定，如果元素的分布极不均匀，在极端情况下，第一个桶中有 n-1 个元素，最后一个桶中有 1 个元素，时间复杂度将退化成 nlogn

3.5. ¶使用场景

外部排序

所谓的外部排序就是：数据存储在外部磁盘，如果数据量比较大，内存优先，无法将数据全部加载到内存中

使用条件苛刻

首先，要排序的数据需要很容易就能划分成 m 个桶，并且，桶与桶之间有着天然的大小顺序。这样每个桶内的数据都排序完之后，桶与桶之间的数据不需要再进行排序。

其次，数据在各个桶之间的分布是比较均匀的。如果数据经过桶的划分之后，有些桶里的数据非常多，有些非常少，很不平均，那桶内数据排序的时间复杂度就不是常量级了。在极端情况下，如果数据都被划分到一个桶里，那就退化为 O(nlogn) 的排序算法了

比如说：

有 10g 的订单数据，希望按照订单金额排序（假设都是正整数），但内存只有 1g，没办法一次性把 10g 的数据全部加载到内存中，这个时候怎么办呢？借助桶排序的思想来处理这个问题

  1. 先扫描一遍文件，看订单金额所处的数据范围。

  2. 假设扫描之后得知金额最大是 10w，最小是 1.

  3. 我们将所有订单根据金额划分到 100 个桶里，第一个同我们存储1-1000 元的订单，第二个桶存储 1001-2000 之间的订单，一次类推。每个桶对应一个文件，并且按照金额范围的大小顺序编号命名（00，01，02、、、）

  4. 理想情况下，如果金额均匀分布，那么订单会被划分到 100 个文件中，每个文件存储大约 100m 的数据，可以将这 100 个小文件，依次放入内存中，用快排来排序

  5. 等所有文件都排序号之后，只需要按照文件编号，从小到大依次读取每个小文件中的订单数据，并将其写入到一个文件中

但是如果金额分布不均匀，大部分金额在 1-1000 这个范围，那么就继续将这个小区间划分为 10 个小区间，1-100.101-200、、、

如果划分后，还是不均匀，那么久继续划分，直到所有的文件都能读入内存为止。
