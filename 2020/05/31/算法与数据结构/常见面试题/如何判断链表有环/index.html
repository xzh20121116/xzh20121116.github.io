如何判断链表有环

阅读量: 今晚打老虎 2020-05-31 20:43:04

1. 题目：如何判断链表有环

有一个单向链表，链表中可能出现“环”，如何判断链表是有环链表呢？

2. 方法 1：徒手遍历

从头结点开始，依次遍历链表中的每一个结点

每遍历一个新节点，就从头检查新节点之前的所有节点，用新节点与之前的所有节点作比较

如果发现当前节点与之前的某个节点相同，说明该节点被遍历过两次，链表有环

2.1. ¶缺点

效率低

3. 方法 2：HashSet 对比

创建一个HashSet 集合，保存节点 id，存储所有遍历过的节点

依次遍历单链表中的每一个节点，每个节点都与结合作对比，如果发现集合总存在，那么说明链表有环

与方法 1 类似，本质区别是使用了 HashSet 做为额外的缓存

3.1. ¶时间复杂度

on，时间上已经最优了

3.2. ¶空间复杂度

使用了额外的存储空间，on

空间上还可以优化

4. 终极方法：

创建两个指针 p1，p2，让他们同时指向链表的头结点。

开始循环，在循环中，让 p1 每次移动 1 个节点，p2 每次移动 2 个节点

比较p1 和p2 的节点是否相同，如果相同，则链表有环

4.1. ¶说明

在一个环形跑道上，两个运动员从同一个地点棋牌，一个快，一个慢，经过一段时间之后，速度快的运动员，必然转上速度慢，因为跑道是圆的。

4.2. ¶时间复杂度

时间复杂度是 on

4.3. ¶空间复杂度

除了两个指针，没有任何的额外空间，空间复杂度是 o1

4.4. ¶示例图

  1. p1、p2 指向头结点

  2. p1 向后移动 1 个，指向 3；p2 向后移动 2 个，指向 7

    对比：发现不相同，继续移动

  3. p1 向后移动 1 个，指向7；p2 向后移动 2 个，指向6

    对比：发现不相同，继续移动

  4. p1 向后移动 1 个，指向2；p2 向后移动 2 个，指向1

    对比：发现不相同，继续移动

  5. p1 向后移动 1 个，指向6；p2 向后移动 2 个，指向6

    对比：发现相同，停

4.5. ¶代码实现

public class 判断链表有环 {

	public static void main(String[] args) {
		Node n1 = new Node(5);
		Node n2 = new Node(3);
		Node n3 = new Node(7);
		Node n4 = new Node(2);
		Node n5 = new Node(6);
		n1.next=n2;
		n2.next=n3;
		n3.next=n4;
		n4.next=n5;
		n5.next=n2;
		System.out.println(isCycle(n1));
	}
	static boolean isCycle(Node head){
		Node p1 = head;
		Node p2 = head;
		while (p2!=null & p2.next!=null){
			p1=p1.next;
			p2=p2.next.next;
			if (p1==p2){
				return true;
			}
		}
		return false;
	}
}

class Node {
	int data;
	Node next;

	Node(int data) {
		this.data = data;
	}
}

5. 扩展题目1：求出环的长度

非常简单

当两个指针首次相遇的时候，让两个指针从相遇点继续往后走，并开始统计走的步数，直到两个指针再次相遇，此时统计出来的步数就是环长

因为指针 p1 每次走 1 步，p2 每次都两步，速度差是 1 步，当两个指针再次相遇时，p2 比 p1 整整多走了 1 圈

环长 = 速度差*前进次数

在 6 相遇了，开始计算步数 p1的位置 p2的位置
第一步 8 1
第二步 1 6
第三步 2 1
第四步 6 6

因此环长是 4

6. 扩展题目 2：求出入环节点

D：从链表头到入环点的距离

s1：从入环点到首次相遇点的距离

s2：从首次相遇点到入环点的距离

两个指针首次相遇的时候，各自走了多少距离呢？

p1一次只有 1 步，因此走过的距离是 D+s1

p2 一次走 2 步，多走了 n 圈（n>=1）因此是 D+s1+n （s1+s2）

因为 p2 的速度是 p1 的 2 倍，所有所走的距离是 p1 的 2 倍

2 （D+s1）=D+s1+n （s1+s2）

经过整理得出

D=(n-1)(s1+s2)+s2

也就是说，head 到入环点的距离，等于首次相遇点绕环 n-1 圈再回到入环点的距离

这样一来，只要把其中一个指针放回到 head，两个指针每次向前走 1 步，他们最终相遇的节点，就是入环点

  4. 指针 p1 和 p2 各自前进一步

  5. 指针 p1 和 p2 各自前进一步

    相遇了，节点 2 就是入环点
